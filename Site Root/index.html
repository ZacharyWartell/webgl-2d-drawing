D3 Beginner Project

For project you will extend the Observable notebook project https://observablehq.com/@d3/connected-scatterplot.

  1. Git Repo
    1. Create a new remote repository on the Git server called "--".
      1. Remember to make your remote repository "Private"
      2. Remember to give the TA and Professor "Reporter" Access.
    2. Now clone your remote repo:
      lucretius@Monad42 ~/ITCS_/
      $git clone https://cci-git.uncc.edu/your_user_id/3120_P2_Planet_Mobile.git 2
      Cloning into 'Project_X'...
      warning: You appear to have cloned an empty repository.

  2. Briefly study the example and code at https://observablehq.com/@d3/connected-scatterplot.

  3. File Organization: Create initial files as follows:
    1. In your project directory, create a subdirectory libs
    2. Download the latest version of D3 from https://d3js.org/.  Copy the D3 source .js files to libs.
    3. Create two files in your project directory index.html and connected-scatterplot.js
    4. Initialize index.html as a basic HTML page that:
      1. loads D3 from directory libs.
      2. loads connected-scatterplot.js.
    5.  Make connected-scatterplot.js import D3 as a JavaScript import module. (Read MDN documentation: https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference/Statements/import if you are not familiar with JS module's).
    6. Add a function main() to connected-scatterplot.js  that prints "Hello World" to the console log.
    7. Git add and commit all files with message "-setup inital file organization"
  4. Port:  Port Observable https://observablehq.com/@d3/connected-scatterplot's D3 code to your JS and HTML files.
    1. Download any referenced data files for https://observablehq.com/@d3/connected-scatterplot.into your project directory.
    2. Port the JavaScript source code from the Observable notebook to your connected-scatterplot.js creating functions and classes as necessary.

      • See the Canvas Announcement Notebook Cell 'auto-magic'
      • The Observable notebook code uses several functions that are not part of D3 or JavaScript.    To port the notebook code make the following substitutions:  instead of Observable FileAttachment read about and use FileReader.onload with an HTML <input> of type="file".
    3. Verify the code works in the Chrome browser
    4. Git add and commit all files with message "-ported connected scatter plot"
  5. Feature:  Pan and Zoom: Add a pan and zooming a user interface.  The input control method is up to you.  That is you can use a set of keyboard keys, or implement a mouse interface or a multi-touch user interface  (see MDN Touch_events).
    • Panning and zooming is done in SVG by adjusting the viewbox (see MDN viewBox
    • For touch the option see MDN Touch_events.
    MDN viewbox)
  6. Feature:  Data Point Selection: Add the ability to hover the the mouse pointer over any data point and display a pop-up window with labeled numerical values of all three of the data points 3 dimensional attributes (year, miles per person per year, cost per gallon).
    Git add and commit with message "-[feature] Data Point Selection"
    See MDN Mouseover Event MDN Popups
  7. Feature:  Time Range Selection: Add a GUI element slider to select an time internal (i.e. a start year and end year). The extreme possible values, min and max, should be determined by the data loaded.   You must choose one of two implementations.
    Git add and commit with message "-[feature] Time Range Selection"
    1. Option I [easy] - use a pair of <input> range elements.  One sets the start year and the other the end year.  For full credit you must also implement the logical that prevents the user from sliding the start year higher than the end year or vice-versa.
      See MDN Input Range
    2. Option II Use SVG drawing and object selection to implement your own interval slider. Yes, add-on libriares provide these, but the extra credit task here is to learn some more SVG and implement you own concept. A example library that demonstrates what you are to create is the jQuery Range slider shown at https://jqueryui.com/slider/#range
  8. Feature:  Time Range Highlighting:  Modify the drawing of the plotted curve to highlight the section of the curve corresponding to the time range selected in the UI above.
    Git add and commit with message "-[feature] Time Range Highlighting"
  9. Feature:  Color Coding Bivariate Delta Values: In addition to the year, each plot point has two data values. As we move from one year to the next these two values can increase, decrease, or stay the same.  Use a the textbook and class we discuss "Bivariate Color Sequences" (4.9.7), that is color coding to bi-variate data.   Pick one of the two recommended methods for bivariate data color coding and color code the plot points using that color coding
    Git add and commit with message "-[feature] Color Coding Bivariate Delta Values"
  10. Feature:  Color Coding Legend:  Add a color coding legend to the page.   See Figure 4.36 in the textbook for an example.
    Git add and commit with message "-[feature] Color Coding Bivariate Delta Values"
  11. Feature:  Generalizing for Other Data Sets: 
    1. Find another data set that you are interested in that is temporal (one dimension is time) and has at least 4 other dimensions that are numerical.    Git add and commit the data file.
    2. Generalize your code so that the user can choose whether to load the original driving data or your new data file and the display adapts accordingly.  This will require generalizing the code that handles step like the axis labeling, that creates the color coding legend, etc. 
    3. Add 4 dropdown menus labeled:  x-axis,y-axis, hue, and chrome or lightness (depending on what choice you made in"Color Coding Bivariate Delta Values".    Each menu lists the four non-temporal dimensions of the new dataset.  Modify the D3 code so that the user's menu choices change which of the four dimension is for your four visualization channels (i.e. x-axis,y-axis, hue, and chrome or lightness).
    4. Git add and commit with message "-[feature] generalized for different datasets"

Appendix II: Academic Integrity

See the course syllabus regarding partial credit and the late penalty policy.  

This is an individual student project.   Each student should be writing and submitting their own code. 

Students can discuss general aspects of the various API's (WebGL, HTML5, etc) with each other and this is strongly encouraged.   Discussing algorithm's at the pseudo-code level is also acceptable.  

However, it is not permissible:

  • to copy code from other students
  • copy code implementing entire functions or entire algorithms from Internet resources
  • to copy code from other students from prior semesters
  • translate an algorithm found on the Internet implemented in programming X and re-write it in language Y.

For example, Objective 4 is to demonstrate the student's ability to understand a geometric computation at the diagrammatic, algebraic and pseudo-code level and then implement code that solves that geometric problem.  Copying code from the Internet that already implements the algorithm (1) does not demonstrate programming ability nor the ability to translate geometric computation from a high-level description to code and (2) constitutes plagiarism if the resource is not cited immediately next to the copied code segment.   For these reasons this would violate academic integrity.

If you have questions about whether using a particular resource is acceptable or not, email the TA and/or Professor.
