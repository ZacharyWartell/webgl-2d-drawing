For this project you will both create a number of small HTML5 pages as well as use the Chrome debug Console to try out a variety of code snippets embedded in the reading.   As with previous tutorials these code snippets will be save to various Console log files which must be saved and git added.

Total number of points for the project is 100.

Coding Standards:

[10 pts] points of your project grade will be based on following good programming practices.   Including the following:
  • consistent indentation, variable naming, etc.
  • commenting practice
    • at least a comment per function, describing it's parameters and function's purpose
  • git commit log
    • There should be at least one descriptive git commit message associated with your achieving each of the numbered requirements (listed below).   Think of your git commit messages as professional journal or blog.
    • Good programming practice would suggest you should be making additional commit messages, as you develop your code and debug it, etc.
    • Remember, you do not need to git push with the same frequency as you git commit.

  1. Line Drawing : [ 0 pts] : In Line Mode, clicking a pair of points will draw a line between those points.  As new vertices are clicked a GL_POINT is drawn to indicate the vertex location.   The sample code already implements this for you.
  2. Triangle Drawing : [10 pts] : In Triangle Mode, clicking a three points will draw a filled triangle using those points.  Similar to Line Mode, as new vertices are clicked a GL_POINT should be drawn to indicate the new vertex location.  After three points are selected the triangle is drawn and the GL_POINTS at the vertices disappear.
  3. Quad Drawing : [10 pts] : In Quad Mode, clicking four point will draw a quadrilateral. Similar to the modes, vertices should be draw for visual feedback.   WebGL does not support quads, but only triangles. The WebGL book Chapter 2 Section “Hello Rectangle (HelloQuad)” describes how to draw a quadrilateral with triangle based primitives.  After three points are selected the triangle is drawn and the GL_POINTS at the vertices disappear.

Part II:

Using the second mouse button, you will be able to select one object. (You may use either the right or middle mouse button, whichever works best in your browser) .

  1. Line Selection : [10 pts] : To determine if a line is selected, implement a function that computes the distance of the mouse click point to each line segment, if the distance is within a few pixels of line L, line L becomes the selected object.

    Detailed Requirements:  Implement the algorithm as described in the following document:
                       https://www.geometrictools.com/Documentation/DistancePointLine.pdf
    Add the code to the file math2D.js in the function called pointLineDist.  

  2. Triangle Selection : [15 pts] :  To determine if a triangle is selected, implement a function to test if a point is inside each triangle.  If mouse the is inside a triangle T, triangle T becomes the selected object.

     Detailed Requirements:  Implement the point-in-triangle test by computing the barycentric coordinate of point p using the triangles vertices to define the barycentric system.   Test the values of the barycentric coordinates of p as discussed in class to test whether p is inside the triangle.    Implement the algorithm as described in class (ITCS-2D Coordinates.pptx and ITCS-Scan_Conversion 2.pptm).   Add the code to the math2D.js function called barycentric().
  3. Quad Selection: [10 pts] :  To determine if a quad is selected, reuse the barycentric coordinate triangle selection, but test again each triangle making up the quad.   If the mouse is inside a quad Q, Q becomes the selected object.
  4. Selected Object Indicator: [5 pts] :  The vertices of the one selected object should be drawn as GL_POINTS in order to visually distinguish that the object is selected.    Once the object is no longer the selected object, these GL_POINTS should disappear.  Use a different color for these points than those used when drawing.
  5. Multiple Objects: [5 pts] :   If multiple objects all pass their selection test, then successive mouse clicks -- made without mouse movement -- should cycle through selecting each of the object's that passed the selection test.   Remember, only one object is considered the selected object at a time, however.

  1. Delete Object : [10 pts] :

Appendix IV: Academic Integrity

See the course syllabus regarding partial credit and the late penalty policy.  

This is an individual student project.   Each student should be writing and submitting their own code. 

Students can discuss general aspects of the various API's (WebGL, HTML5, etc) with each other and this is strongly encouraged.   Discussing algorithm's at the pseudo-code level is also acceptable.  

However, it is not permissible:

  • to copy code from other students
  • copy code implementing entire functions or entire algorithms from Internet resources
  • to copy code from other students from prior semesters
  • translate an algorithm found on the Internet implemented in programming X and re-write it in language Y.

For example, Objective 4 is to demonstrate the student's ability to understand a geometric computation at the diagrammatic, algebraic and pseudo-code level and then implement code that solves that geometric problem.  Copying code from the Internet that already implements the algorithm (1) does not demonstrate programming ability nor the ability to translate geometric computation from a high-level description to code and (2) constitutes plagiarism if the resource is not cited immediately next to the copied code segment.   For these reasons this would violate academic integrity.

If you have questions about whether using a particular resource is acceptable or not, email the TA and/or Professor.
