Part I:

The skeleton code contains a file math2D.js. In contains partial implementation for a number of classes and functions. For Part I, in class Mat2 and Vec2 finish implementing the unimplemented class methods that contain the comment /* \todo needs to be implemented */. Verify these methods work correctly before proceeding to Part II.
  1. class Mat2 - implement the remaining methods of this class
  2. class Vec2 - implement the remaining methods of this class
  3. Verify and Test - In file 2D Drawing.js, find the if-then statement associated with boolean variable justRunMathTest. Temporarily change the justRunMathTest in order to run the test function math2d_test(). Study the code in math2d_test(). Use this code to test your implementation of the Mat2 and Vec2 methods. Add any additional testing code to math2d_test() that you find necessary.

Part II:

Implement the following functionality by clicking with the mouse button.

  1. Line Drawing In Line Mode, clicking a pair of points will draw a line between those points.  As new vertices are clicked a GL_POINT is drawn to indicate the vertex location.   The sample code already implements this for you.

  2. Triangle Drawing : In Triangle Mode, clicking a three points will draw a filled triangle using those points.  Similar to Line Mode, as new vertices are clicked a GL_POINT should be drawn to indicate the new vertex location.  After three points are selected the triangle is drawn and the GL_POINTS at the vertices disappear.

  3. Quad Drawing In Quad Mode, clicking four point will draw a quadrilateral. Similar to the modes, vertices should be draw for visual feedback.   WebGL does not support quads, but only triangles. The WebGL book Chapter 2 Section “Hello Rectangle (HelloQuad)” describes how to draw a quadrilateral with triangle based primitives.  After three points are selected the triangle is drawn and the GL_POINTS at the vertices disappear.

Part III:

Using the second mouse button, you will be able to select one object.(You may use either the right or middle mouse button, whichever works best in your browser).

  1. Line Selection To determine if a line is selected, implement a function that computes the distance of the mouse click point to each line segment, if the distance is within a few pixels of line L, line L becomes the selected object.

    Detailed Requirements:  Implement the algorithm as described in the following document:
                       https://www.geometrictools.com/Documentation/DistancePointLine.pdf
    Add the code to the file math2D.js in the function called pointLineDist.  

  2. Triangle Selection To determine if a triangle is selected, implement a function to test if a point is inside each triangle.  If mouse the is inside a triangle T, triangle T becomes the selected object.

    Detailed Requirements:  Implement the point-in-triangle test by computing the barycentric coordinate of point p using the triangles vertices to define the barycentric system.   Test the values of the barycentric coordinates of p as discussed in class to test whether p is inside the triangle.    Implement the algorithm as described in class (ITCS 4120-2D Coordinates.pptx and ITCS 4120-Scan_Conversion 2.pptm).   Add the code to the math2D.js function called barycentric().

  3. Quad Selection: :  To determine if a quad is selected, reuse the barycentric coordinate triangle selection, but test again each triangle making up the quad.   If the mouse is inside a quad Q, Q becomes the selected object.

  4. Selected Object IndicatorThe vertices of the one selected object should be drawn as GL_POINTS in order to visually distinguish that the object is selected.    Once the object is no longer the selected object, these GL_POINTS should disappear.  Use a different color for these points than those used when drawing.

  5. Multiple Objects If multiple objects all pass their selection test, then successive mouse clicks -- made without mouse movement -- should cycle through selecting each of the object's that passed the selection test.   Remember, only one object is considered the selected object at a time, however.

Part IV:

  1. Delete Object
    1. Add a Delete button.   Pressing the delete button should delete only the one selected object.  (If none is selected, the button does nothing.)
  2. Color Change
    1. Use three sliders with labels Red, Green, Blue.   The their values should range between 0 and 1. 
    2. When a object is selected, that the sliders' values should immediately change to be based on the object type's current color
    3. If you move a slider, the color of the selected object should change based on the slider values

    Comments:  The resources listed earlier give a reasonable introduction to event handlers in HTML5 and JavaScript.   Sliders are implemented with <input type="range"> HTML elements.  The skeleton code provides the 3 sliders and initializes basic event handlers.  You will have to modify these as well as set the range's min and max, etc.   (https://www.w3.org/wiki/HTML/Elements/input/range)
