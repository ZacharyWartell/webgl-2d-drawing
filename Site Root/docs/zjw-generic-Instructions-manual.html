Goals

  1. Best practices for HTML based instruction documents:  Provide a "template" for an HTML5 document which follows best-practices for presenting written instructions for typical computer science assignments.   These features include the following (see also the instruction audience's manual Help.html):
    1. Support for <span> classes that visually designate simple content such as file names, code variable name, function names, user input, etc.
    2. Support for <span>,<div> and <table> classes that designate complex structured content such as
      1. Shell script (CLI) code with annotations
      2. Zoomable screen captures
      3. Embedded video with associated written instructions whose written steps each include a button that advanced the video to the video segment illustrated that written step.
    3. Rubric Generation: Support <span> classes with HTML data- attributes that encode the points different exercises in the assignment are worth and automatically generate a rubric <table> extracting these aattributes from the document's instructions.
    4. Document Variables: Support automatic string replacement of named "document variables" in the document text with text strings specific to the assignment.   For example, the shell CLI examples may all display a path name that user is working in.   The path name text often is specific the assignment name. The HTML written for displaying the shell CLI examples will refer to a "document variable" that will be replaced by a designated text string defined globally in the assignments meta data.
  2. The template is git repo that the user clones and then redirects the origin to their own repo.
  3. HTML5 document should be a static web page, but otherwise the template user must be able to include any HTML5 content -- SVG, WebGL,Canvas, etc. -- in their instruction document.   (This is where relying on Content Management Systems, Learing Management Systems, etc. -- instead of directly coding in HTML5 coding -- always fall short and fails to meet my personal requirements).
  4. Code modules provided with the template (called "template sub-modules") are distributed as git submodule in the template repo.
  5. Template users alter the content of their instruction document without needing to change the code modules provided by the template.
  6. Template users may contribute updates, improvements, bug fixes, to the template's code modules via the git submodule mechanism.

  7. [FUTURE]:
    1. Provide functionally that allows common HTML5 editors such as BlueGriffon instantied the various components listed in #1 using the HTML5 <template> mechanism rather than copy and pasting example HTML code.
    2. ? If (a) is no feasibly in a short amount of time, provide a simple builtin HTML5 editor based on, that is augmented with GUI that can insert the <template>'s, and format selected text with with specialized <span> classes discussed in 1.a

  8. [CAVEATS]

    1. Only support the Chrome browser: All my course HTML5 based projects require the student varify their assigment's code run in Chrome.  This is the target browser the TA uses for testing all assignments.  Neither I, nor students in most CS courses, have time to address all the madness of cross-browser compatibility.    That type of code development requirement is typically only appropriate in specialied courses in "Web Application Developement" or perhaps "Software Engineering"..

Usage

Version "April"

  1. Create an empty repo on gitlab.   Assume repo's clone URL is Your-Repo
  2. Clone and lightweight fork https://gitlab.com/zwartell/zjw-generic-instructions-template into a new repo for you new assignment.
    $ git clone https://gitlab.com/zwartell/zjw-generic-instructions-template Your-Repo-Directory
    $ git remote rename origin upstream Your-Repo
    $ git push origin
    $ git submodule update --remote zjw-generic-instructions
  3. Customize the following standard components of the ./index.html
    1. <title></title>
  4. Customize the following standard elements of the ./content.html
    1. <div id="AssignmentName" class="Data"z
      Documentationfor each field is in the class AssignmentName in ../ClassAssignmentWartell.ts
  5. Develop your new assignments specific content in content.html using the following protocol:
    1. \todo elaborate
  6. If you make updates, bug fixes, improvements relevant to shared subodule, zjw-generic-instructions, use following protocol
    1. \todo elaborate
